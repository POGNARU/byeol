# 별 (byeol) — 밤하늘 서고

포근나루 / 디퍼 OS의 **밤하늘(nightsky) 아카이브**. 한 주 동안 길어 올린 씨앗·오아시스 문장이 별이 되어 뜨는 밤하늘 대시보드를 한자리에 모은다.

- **공개 링크**: https://pognaru.github.io/byeol/
- **공개 범위**: 링크 아는 사람만 (검색 차단 — 모든 페이지 `noindex` + `robots.txt Disallow /`)
- **호스팅**: GitHub Pages (POGNARU/byeol, main 브랜치)

## 구조

```
byeol/
├─ index.html          # 서고 표지 (밤하늘 갤러리 — 2026 주간 선반 + 지나온 해들 선반)
├─ entries/            # 각 밤하늘 HTML
│  ├─ nightsky_YYYY-MM-DD.html   # 주간 (2026~, 파일명은 생성일 · 범위는 전주 월~일)
│  └─ nightsky_YYYY.html         # 연간 (2023·2024·2025 — 한 해의 별자리 한 장)
├─ robots.txt          # 검색엔진 전면 차단
├─ .nojekyll           # Jekyll 처리 비활성화
└─ README.md
```

## 과거 새벽의 별자리 (2026-07-10 추가)

호수(분석 식구)가 몽롱쓰기 보관함 2023~2025 전체(265파일)와 2026년 1~4월 모닝페이지를 통독해 별자리를 소급했다.

- **연간 3장** — `nightsky_2023.html`(오리온·도전) / `nightsky_2024.html`(북두칠성·방향) / `nightsky_2025.html`(왕관자리·회복). 과거 해는 한 해를 별자리 하나로 묶는다(2026-07-10 햇살 확정).
- **2026 주간 23장** — 1/12~6/28 (월~일 주). 5월 첫 주~6월 28일 8주는 2026-07-10 햇살 요청으로 백필 완료(바다가 모닝페이지 통독 후 생성 — 오리온·북두칠성·왕관·카시오페이아·백조를 주마다 다르게 배치). 6/29 주부터는 매주 사막여우의 "밤하늘" 루틴이 이어간다.
- **씨앗은 전부 재구성 문장** — 원문을 그대로 옮기지 않고 뜻만 살려 다시 썼다(사적 새벽 글 보호, 실명 `○○`). 씨앗 원장은 디퍼 살롱 볼트 `밤하늘 씨앗/` 폴더(연도별·주간별 md).
- 생성 스크립트(도안집 규격 반영 파이썬)는 세션 스크래치패드에서 사용 — 페이지는 정적 HTML이라 재생성 없이 그대로 유지된다.

## 새 밤하늘 추가하는 법 (2026-07-19 햇살 확정 — 여기까지가 한 흐름이다)

"밤하늘 지어줘"는 **디퍼 살롱에 이미 있는 씨앗만 취합하라는 뜻이 아니다.** 아래 다섯 걸음을 끝까지 밟는다.

1. **볼트 원문을 먼저 읽는다** — `Obsidian Vault/모닝 페이지/`와 `Obsidian Vault/별지기 저널/`에서 그 주(월~일) 글 전체. 원본은 읽기만 하고 옮기지 않는다(사적인 새벽 글 보호).
2. **씨앗을 발췌해 디퍼 살롱에 저장한다** — 글 하나당 하나, 저장 전 후보를 보여 확정받는다. 위치는 `deeper-salon/01-Grammar/Insights/`.
   ⚠️ **저장 전에 Insights 폴더를 먼저 확인할 것** — 냠냠·살롱 실습으로 이미 씨앗이 나와 있을 수 있다. 출처만 읽고 목적지를 안 보면 중복이 난다(2026-07-19 '두 욕망' 사고).
3. **그 주 씨앗을 전부 취합해** 대시보드를 만든다 (실명은 `○○`로 가림). 새로 뽑은 것 + 이미 있던 것 모두.
4. 완성된 `nightsky_YYYY-MM-DD.html`을 `entries/`에 복사하고(`<meta name="robots" content="noindex, nofollow">` 포함), `index.html`의 `.grid` 맨 위에 카드 한 장을 추가한다 (최신이 위).
5. **마무리는 언제나 3종세트** — `git add -A && git commit && git push` + README 갱신. GitHub Pages가 자동 배포한다.

## 별·별자리 디자인 (2026-07-09~)

밤하늘의 별과 별자리 규격은 볼트의 **`deeper-salon/00-Daily/밤하늘/_밤하늘_도안집.md`** 가 원본이다.

- **예쁜 별** — 4각 스파클 별 + 발광(glow) + 반짝임 애니메이션. 가장 밝은 2~3개엔 십자 빛줄기(glint). 북극성도 사각 스파클.
- **매번 다른 별자리** — 그 주의 결/상징으로 북두칠성·카시오페이아(W)·백조·오리온·왕관자리 중 하나. 늘 같은 모양이 되지 않게.
- **씨앗은 모두 하늘에** — 대표 씨앗만 별자리로 잇고, 나머지는 금빛 잔별로 곁에 함께 띄운다.
- **깊은 밤 배경** — `#04120C→#081E14→#0C2A1C`.

첫 적용: `entries/nightsky_2026-07-04.html` (북두칠성). 이후 매주 이어짐 — 최신: `entries/nightsky_2026-07-19.html` (7월 13–19일, 오리온 · 별 23개).

## 어두운 문장도 별이다 (2026-07-19 햇살 확정)

밤하늘은 기본적으로 아픈 문장을 싣지 않는다(사막여우 규칙). 다만 **햇살이 올려달라고 하면 원문 그대로 올린다** — "어두운 것도 내게 있는 것들이니." 씨앗으로 다듬지 않은 원문 인용은 `✧`(속 빈 별)로, 다듬은 씨앗은 `✦`(찬 별)로 구분하고 메타에 `· 원문 그대로`를 붙인다. 하늘에서는 구분 없이 같은 금빛 별로 뜬다.

## 원칙

- **원본은 볼트에.** 밤하늘 원본은 `deeper-salon/00-Daily/밤하늘/`에 있고, 여기엔 실명을 가린 공개본만 둔다.
- **실명 가림.** 등장 인물 실명은 `○○`, 제목은 "나의 밤하늘". 볼트 원본 씨앗은 손대지 않는다.
- **제작 범위 = 전주 월~일.** "밤하늘 만들어줘"는 언제 요청하든 **지난주(월~일)**가 재료 범위다. 예외는 **일요일에 요청한 경우**뿐 — 이때만 당주(월~일)를 담는다.
- **파일명 = 생성일.** 파일명은 만든 날짜(`nightsky_YYYY-MM-DD.html`)로 둔다. 일요일 전에 미리 만들어도, 나중에 업데이트가 필요한지 날짜로 바로 알아보기 위함이다.

## 파비콘

브라우저 탭 아이콘으로 포근나루 심볼(나룻배)을 쓴다. 2026-07-22 적용, 같은 날 심볼을 새 시안으로 교체하며 다시 뽑았다.

```
favicon-32.png    탭 아이콘
favicon-180.png   애플 터치 아이콘
```

레포 루트에 두고, 모든 HTML의 `<head>`에 상대경로로 링크한다. 하위 폴더 문서는 `../favicon-32.png`처럼 앞에 `../`를 붙인다 — GitHub Pages 프로젝트 사이트는 `/레포명/` 아래로 서빙되므로 `/favicon-32.png` 같은 루트 절대경로를 쓰면 안 된다.

원본은 볼트 `클로드 에이전트/브랜드/로고/`에 있다. 심볼을 바꾸면 거기서 다시 뽑아 이 레포의 두 파일을 교체한다.
